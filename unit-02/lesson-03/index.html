02

개념설명

⏱ 15분▾

문자 → 이진수: 아스키 코드(ASCII)

컴퓨터는 문자를 직접 저장할 수 없기 때문에, 각 문자에 고유한 숫자를 부여합니다. 아스키 코드는 영문자·숫자·특수문자 128개를 7비트(0~127)로 약속한 표준입니다.

  • A = 65₁₀ = 01000001₂
  • a = 97₁₀ = 01100001₂
  • 0 = 48₁₀ = 00110000₂
  • 유니코드: 한글·한자·이모지 포함, 전 세계 문자 표준
⚠️
오개념: 유니코드는 아스키코드의 확장입니다. 한글 '가'는 아스키코드에 없고, 유니코드에서 AC00₁₆(44032₁₀)으로 표현됩니다.

이미지 → 픽셀 + RGB

디지털 이미지는 수많은 작은 점인 픽셀(Pixel)로 구성됩니다. 각 픽셀은 빨강(R)·초록(G)·파랑(B) 세 색의 밝기(0~255)로 색상을 표현합니다.

  • 해상도: 가로×세로 픽셀 수. 많을수록 선명
  • RGB(255,0,0): 빨간색 / RGB(0,0,0): 검정
  • 24비트 색상: 픽셀 1개 = R·G·B 각 8bit = 3byte
픽셀 = 색상 데이터
⚠️
오개념: 해상도는 화면 크기가 아닙니다. 같은 크기 화면이라도 픽셀 수가 많으면(고해상도) 더 선명합니다.

소리 → 샘플링

소리는 연속적인 아날로그 파형입니다. 이를 디지털로 변환하려면 일정 간격으로 파형의 값을 기록하는 샘플링(Sampling)을 합니다.

  • 샘플링 주파수(Hz): 1초에 측정하는 횟수. 높을수록 고음질
  • CD 품질: 44,100 Hz (44.1kHz)
  • 비트 깊이: 각 샘플의 정밀도. 16bit = 65,536단계
⚠️
오개념: 샘플링 주파수가 낮으면 원래 소리의 정보가 손실됩니다. 나이퀴스트 정리: 원음 최고 주파수의 2배 이상으로 샘플링해야 합니다.

영상 → 프레임 + FPS

동영상은 정지 이미지(프레임)를 빠르게 연속 재생하는 원리입니다. FPS(Frame Per Second)는 초당 프레임 수입니다.

  • 영화: 24 FPS
  • TV·게임: 30~60 FPS
  • 코덱(Codec): 영상 압축·해제 알고리즘 (H.264, H.265, VP9)
  • 해상도: 720p, 1080p(FHD), 4K(UHD)
