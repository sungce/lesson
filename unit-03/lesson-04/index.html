02

개념설명

⏱ 15분▾

알고리즘 분석 — 효율성 비교

같은 문제도 알고리즘에 따라 실행 시간과 자원 사용이 달라집니다. 더 빠르고 자원을 적게 쓰는 알고리즘이 더 효율적입니다.

4가지 알고리즘 설계 전략 (교과서 pp.104~106)

🔀 문제 나누기
큰 문제 → 작은 문제로 분할하여 각각 해결
장점: 복잡한 문제 쉽게 해결
단점: 분할·합치는 알고리즘 추가 필요
🔁 시행착오 반복
가능한 모든 경우를 시도
예: 자물쇠 00→01→02→...
장점: 단순
단점: 경우 많으면 매우 느림
↩️ 되추적
막히면 이전 단계로 돌아가 다른 길 탐색
예: 미로 탈출
장점: 반드시 해결 가능
단점: 느릴 수 있음
⭐ 최선 방법 선택
매 순간 가장 좋아 보이는 선택
예: 등산 — 쉬운 길 선택
장점: 빠름
단점: 최적 보장 안 됨
⚠️ 오개념: '최선 방법 선택'은 매 순간 최선이지만 전체적으로는 최적이 아닐 수 있습니다. 예: 거스름돈 문제에서 동전 종류에 따라 실패할 수 있습니다.
03

인터랙션 — 알고리즘 전략 매칭

⏱ 15분▾
💡 각 문제 상황에 가장 적합한 알고리즘 전략을 선택해보세요!
🎯 상황 1
시험지 29장을 학생 번호 순서대로 정리하려고 한다. 10번대 단위로 먼저 묶은 뒤 각 묶음을 정리하는 방법을 사용한다.
🎯 상황 2
잠근 자물쇠의 두 자리 비밀번호를 잊어버렸다. 00부터 시작해서 자물쇠가 열릴 때까지 번호를 1씩 늘려가며 시도한다.
🎯 상황 3
미로 탈출 퍼즐을 풀고 있다. 막다른 길이 나오면 이전 갈림길로 돌아가 다른 방향으로 이동한다.
효율적인 알고리즘 설계가 중요한 이유로 가장 적절한 것은?
①알고리즘의 이름을 더 짧게 만들 수 있기 때문에
②실행 시간을 줄이고 컴퓨터 자원·에너지를 절약할 수 있기 때문에
③어떤 프로그래밍 언어로도 코딩할 수 있기 때문에
④순서도에서 사용하는 기호 수를 줄일 수 있기 때문에
